What is a navigation message

A satellite navigation message is a message broadcast by a navigation satellite to the user describing the parameters of the navigation satellite's operation status, including the system time, ephemeris, ephemeris, correction parameters of the satellite clock, the health of the navigation satellite and the ionospheric delay model parameters. The parameters of the navigation message provide time information to the user, and the position coordinates and velocity of the user can be calculated using the parameters of the navigation message

Structure of a navigation message

Each frame is 1500 bits long and has a duration of 30 s. Each frame consists of 5 subframes, each of which is 300 bits long and has a duration of 6 s. Each subframe consists of 10 words, each of which is 30 bits long. The first two words of each subframe are telemetry word (TLW) and handover word (HOW), and the last 8 words are data blocks. Telemetry word (TLM)

Of the 30 bits of the telemetry word, the first 8 bits have a fixed value of 10001011 sync code, 9-22 bits are telemetry codes containing telemetry information, 23 and 24 bits are reserved, and the last 6 bits are parity codes. The synchronization code is used to match the received data code, to realize the search to determine the starting position of the subframe, and to provide the conditions for subsequent unframing.

Handover word (HOW)

Immediately after the telemetry word, bits 1-17 of the handover word are the truncated intraperiodic count values obtained on the Z counter. The 18th bit is the warning flag, when the flag is 1, warning that the URA value provided by the first data block of this satellite may be larger than its true value, and the 19th bit is the A-S flag, when the flag is 1, indicating that anti-electronic spoofing measures are applied to this satellite. Bits 20-22 are the subframe identification flags, corresponding to 001-101, which are used to identify which subframe this is. Bits 23 and 24 are the values that can be calculated to make the last 6 bits of the parity code end with 00. The last 6 bits are the parity bits.

First data block

The first data block includes 3-10 words of the 1st subframe, for time information, etc., providing the satellite's time correction parameters, health status, etc.

  • Number of days of the week, weekly week value + 1, maximum value is 1023, if it exceeds 1023 it will return to zero.
  • User Range Progress Accuracy (URA), a statistical value for all errors caused by the GPS ground monitoring part and the space constellation part, varies from 0 to 15, the larger the URA value, the less accurate the GPS distance measurement value obtained from this satellite.
  • Satellite health status: 6 bits in total, the highest bit 0 indicates that the navigation message is correct, 1 indicates that there is an error in the navigation message, and the last 5 bits indicate different error states.
  • Clock correction parameter: used to correct the clock difference of the satellite clock.
  • Group delay correction value: The single frequency receiver uses this value to correct the group delay.
  • Clock data period number: 10 bits of data to indicate the "period number" of the clock data block, telling the receiver to process the clock correction parameters in the data only when the clock correction data is updated.

Second data block

The second data block, consisting of subframes 2 and 3, provides the satellite ephemeris parameters that describe the spatial position and speed of the satellite at each moment. A set of ephemeris is valid for 4 hours, and the 8-bit ephemeris data period number, which is broadcasted in both subframe 2 and 3, is used to determine if it has expired.

Third data block

The third data block consists of frames 4 and 5, which broadcast the ephemeris parameters of all satellites, ionospheric correction parameters, GPS time vs. UTC time, satellite health status, etc. The third data block consists of frames 4 and 5, which broadcast the ephemeris parameters of all satellites, ionospheric correction parameters, GPS time vs.
Subframes 4 and 5 of a frame do not have enough space to broadcast the ephemeris parameters of all satellites and are paged.

  • Pages 2,3,4,5,7,8,9,10 of subframe 4 provide the ephemeris of satellites with PRN numbers 25-32.
  • Page 18 of subframe 4 provides the ionospheric correction parameters and the parameters of the relationship between GPS time UTC.
  • Page 25 of subframe 4 provides the satellite health status for satellites with PRN numbers 25-32.
  • Page 1-24 of subframe 5 provides the ephemeris for satellites with PRN numbers 1-24.
  • Page 25 provides the health status of satellites with PRN numbers 1-24, the calendar reference time and the calendar reference week number.
