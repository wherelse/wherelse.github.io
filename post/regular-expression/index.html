What is a Regular Expression

Regular expressions use a single string to describe and match a series of strings that match a syntactic rule. By using regular expressions, you can.

  • Test for patterns within a string. You can test the input string to see if a phone number pattern or credit card number appears within the string for data validation.
  • Replace text. Regular expressions can be used to identify specific text in a document, remove that text entirely or replace it with other text.
  • Extract substrings from strings based on pattern matching. Can find specific text within a document or in an input field.

Basic Matching Syntax Characters

The following example uses Regular expression as an example

Character Usage Example
* matches the preceding subexpression zero or more times es* matches e or ess
+ matches the preceding subexpression one or more times es+ matches es or ess, but not e
. .e matches Re and re, which means: any character followed by a letter e
? is used to indicate that the previous character is optional, matching the preceding subexpression zero or once es? matches e or es, but not ess
^ check if the matching character is the starting character ^re matches re at the beginning of regular but not in expression, in this case we ignore case issues
$ check if the matching character is the last character r$ matches r at the end of regular, but not the other r's
( ) match the characters inside the brackets in exact order (re) matches only re, not others
[ ] match any character contained in square brackets, no order required [rR] matches both uppercase and lowercase
[^ ] match any character not contained in square brackets, no order required [^rR] will match to egular expession
{n,m} match the character before the brackets at least n times, but not more than m times 9{2,4} means match at least 2 digits but not more than 4 digits of the number 9
\ escape character, used to escape the next character \$ is the original symbol $

Abbreviated matching characters

More complex matching statements can be abbreviated

character usage
\w Matches all alphabetic and numeric characters: [a-zA-Z0-9_]
\W Matches non-alphabetic and numeric characters: [^\w]
\d` Matching numbers: [0-9]
\D Match non-numeric: [^\d]

Using the | or branching feature

Similar to some languages, regular expressions can use | to represent or. This can be achieved by using (r|R)e to match case, or by using (c|m|f)at to match cat, mat, and fat at the same time.

(c|m|f)at ==> a fat cat sleep at a mat

Use ? for optional matching

Use (c|m|f)?at to achieve simultaneous matching of cat, mat, fat and at, i.e. the content before the question mark is unmatchable.

(c|m|f)at ==> a fat cat sleep at a mat

Non-printing character matching table

character usage
\f Matches a page break character.
\n Matches a line feed character.
\r Match a carriage return character.
\s matches any whitespace character, including spaces, tabs, page breaks, etc. Note that Unicode regular expressions will match full space characters.
\S Match any non-whitespace character.
\t matches a tab character.
\v Match a vertical tab.
\b Matches a word boundary, i.e. the position between the word and the space.
